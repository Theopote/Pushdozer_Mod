# Pushdozer多人联机功能说明

## 设计理念

Pushdozer是一个**地形处理工具**，在多人服务器上的设计理念是：

1. **工具开放性**: 所有玩家都可以使用pushdozer工具进行地形编辑
2. **配置独立性**: 每个玩家可以有自己独立的配置设置
3. **操作同步性**: 所有地形修改会实时同步给其他玩家

## 核心特性

### 🔓 无权限限制
- 任何玩家都可以使用pushdozer工具
- 无需OP权限或特殊设置
- 真正的"工具"定位，而非"管理员功能"

### ⚙️ 个人配置
- 每个玩家的工作模式、几何体类型、操作半径等设置都是独立的
- 配置保存在客户端，不会影响其他玩家
- 玩家可以根据自己的需求和喜好调整设置

### 🌐 实时同步
- 所有地形修改操作会立即同步到所有在线玩家
- 使用优化的网络传输，支持大规模操作
- 撤销/重做操作也会同步显示

### 🛡️ 合理限制
- 操作半径限制（最大100格）防止恶意破坏
- 可扩展的区域保护系统接口
- 服务器管理员可以集成第三方保护插件

## 使用场景

### 建筑合作
- 多个玩家可以同时使用不同的工具模式协作建设
- 一个玩家挖掘，另一个玩家平整，第三个玩家装饰

### 地形创作
- 每个玩家可以使用自己喜欢的工具设置
- 实时看到其他玩家的创作过程
- 支持团队创作大型地形项目

### 服务器建设
- 适合创造服务器和建筑服务器
- 玩家可以自由使用地形工具美化环境
- 管理员可以通过区域保护控制重要区域

## 技术优势

### 网络优化
- 批处理系统减少网络负担
- 智能分块传输支持大规模操作
- 异步处理不影响游戏性能

### 扩展性
- 可集成LuckPerms等权限插件
- 支持WorldGuard等区域保护
- 模块化设计便于自定义

### 稳定性
- 每个玩家独立的撤销历史
- 错误恢复机制
- 并发安全的数据处理

## 对比传统设计

| 特性 | 传统权限模式 | Pushdozer开放模式 |
|------|-------------|------------------|
| 使用权限 | 仅OP/管理员 | 所有玩家 |
| 配置管理 | 全局统一 | 个人独立 |
| 使用便利性 | 需要权限申请 | 即用即有 |
| 协作效率 | 有限 | 高效 |
| 服务器负担 | 管理员压力大 | 分布式使用 |

## 总结

Pushdozer的多人联机设计体现了"工具化"而非"特权化"的理念，让每个玩家都能享受地形编辑的乐趣，同时通过合理的技术限制确保服务器的稳定和安全。这种设计特别适合注重创造和建设的服务器环境。

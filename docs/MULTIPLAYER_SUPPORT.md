# Pushdozer多人联机支持文档

## 概述

Pushdozer模组现已完全支持多人联机游戏。作为一个地形处理工具，每个玩家都可以使用它来修改地形，并且每个玩家可以根据自己的喜好设置独立的配置。

## 主要功能

### 1. 地形操作同步
- 所有地形修改操作（挖掘、放置、平滑等）会实时同步到所有在线玩家
- 使用优化的批处理系统减少网络延迟
- 支持大规模地形操作的分块传输

### 2. 开放权限
- **所有玩家都可以使用pushdozer工具**进行地形编辑
- 操作范围限制（最大半径100格）防止恶意大范围破坏
- 在单人游戏中自动允许所有操作
- 可扩展的区域保护系统接口

### 3. 独立配置
- **每个玩家的配置是独立的**，可以根据个人喜好设置
- 工作模式、几何体类型、操作半径等都是个人设置
- 不会影响其他玩家的配置
- 配置保存在本地 `config/pushdozer_config.json`，修改后会通过 `ConfigSyncPayload` 同步到服务端供你的操作使用，但**不会广播给其他玩家**，也不会写入他人的本地文件

### 4. 撤销/重做系统
- 每个玩家独立的撤销/重做历史
- 撤销操作会同步到所有客户端显示正确的地形变化
- 支持跨世界的操作记录

## 网络协议

### 数据包类型

#### 1. TerrainOperationPayload
用于同步地形修改操作到所有客户端
- 操作类型（挖掘、放置等）
- 影响的方块位置列表
- 新的方块状态列表

#### 2. ConfigSyncPayload
用于验证配置信息（保留接口，未来可能用于扩展）
- 工作模式
- 几何体类型
- 放置模式
- 高度模式
- 操作半径
- 锁定高度
- 选中的自然方块ID

#### 3. UndoRedoPayload
用于撤销/重做操作
- 操作类型（撤销或重做）

#### 4. PermissionCheckPayload
用于权限验证
- 操作类型
- 中心位置
- 操作半径

## 性能优化

### 批处理系统
- 小操作（≤50方块）立即发送
- 大操作使用批处理队列，减少网络包数量
- 智能分块传输，每批最多500方块
- 100毫秒的批处理延迟平衡性能和响应性

### 内存管理
- 使用并发安全的数据结构
- 自动清理已完成的批处理任务
- 守护线程处理网络操作，不阻塞游戏线程

## 服务器配置

### 权限设置
**Pushdozer是一个开放工具，所有玩家都可以使用**：

1. 无需特殊权限设置，所有玩家默认可用
2. 服务器管理员可以通过区域保护插件限制特定区域
3. 可以通过修改代码集成自定义权限系统

### 性能调优
- `MAX_BATCH_SIZE`: 批处理最大方块数（默认500）
- `BATCH_DELAY_MS`: 批处理延迟毫秒数（默认100）
- 操作半径限制（默认最大100格）

## 故障排除

### 常见问题

#### 1. 玩家无法使用工具
**症状**: 工具不响应或显示错误
**解决方案**: 
- 检查是否在区域保护范围内
- 检查操作半径是否超过限制（100格）
- 重新登录服务器

#### 2. 操作不同步
**症状**: 其他玩家看不到地形更改
**解决方案**:
- 检查网络连接
- 重启客户端/服务器
- 查看服务器日志中的网络错误

#### 3. 性能问题
**症状**: 大规模操作造成延迟
**解决方案**:
- 减少操作半径
- 降低批处理大小
- 使用分批操作而非一次性大操作

### 日志等级
- DEBUG: 详细的网络包传输信息
- INFO: 重要的操作和配置更改
- WARN: 权限违规和异常操作
- ERROR: 网络传输失败和严重错误

## 开发者信息

### 扩展权限系统
可以集成第三方权限插件（如LuckPerms）：

```java
private static boolean checkTerrainOperationPermission(ServerPlayerEntity player, 
                                                     String operationType, 
                                                     BlockPos centerPos, 
                                                     int radius) {
    // 在这里添加自定义权限检查逻辑
    // 例如集成LuckPerms或其他权限系统
    return player.hasPermissionLevel(2);
}
```

### 自定义网络包
要添加新的网络包类型：

1. 创建Payload记录类
2. 在NetworkManager中注册
3. 在ClientNetworkHandler中添加处理器
4. 更新相关的操作代码

### 性能监控
服务器管理员可以监控以下指标：
- 批处理队列大小
- 网络包发送频率
- 平均操作延迟
- 内存使用情况
